# Changelog 2025-11-20 (Hotfix 4)

## 🐛 錯誤修復：請求超時問題

### 問題描述
1.  **請求超時**：Node.js 預設的 `fetch` 超時時間可能短於 Python 伺服器首次加載模型所需的時間（特別是在冷啟動時）。
2.  **語法錯誤**：在嘗試修復超時問題時，引入了一個語法錯誤（`try...catch...finally` 結構不完整）。

### 修復內容
-   **延長超時時間**：
    -   `/recognize` 請求：設置為 **5 分鐘**，以確保有足夠時間進行模型加載和推理。
    -   `/crop` 請求：設置為 **1 分鐘**，雖然現在是輕量級操作，但保留緩衝以防萬一。
-   **修復語法**：修正了 `server/routes.ts` 中的 `try...catch...finally` 區塊結構，確保資源正確釋放且錯誤被捕獲。

### 驗證
-   部署後，首次請求（觸發模型加載）應不再因超時而失敗。
-   應用程式應能穩定運行，不會因語法錯誤而崩潰。
