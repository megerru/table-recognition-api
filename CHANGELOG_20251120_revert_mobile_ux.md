# Changelog 2025-11-20 (Revert Mobile UX)

## ↩️ 回滾：手機端框選邏輯

### 變更描述
應用户要求，回滾了關於手機端框選確認的 UX 修改。

### 回滾內容
1.  **RegionSelector 組件**：
    -   移除了 `isLoading` prop。
    -   移除了按鈕上的 Loading 狀態顯示。
    -   移除了 `Loader2` 圖示。
2.  **Home 頁面**：
    -   恢復了 `!recognizeRegionsMutation.isPending` 的條件判斷，即在識別過程中會隱藏框選組件。
    -   恢復了 `!previewData` 的條件判斷，即在顯示結果時隱藏框選組件（雖然這部分邏輯在之前的修復中是為了避免同時顯示，回滾後可能會有重疊或閃爍，但遵從用戶要求恢復到前一版本）。

### 影響
-   手機端點擊確認後，畫面可能會再次出現「空白」或無反應的現象（這是之前的已知問題），直到識別完成。
-   恢復到用戶指定的「前一個版本」行為。
